# 길드 아카이브 디자인 가이드

## 디자인 원칙

- 전투 중에도 빠르게 읽히도록 정보 밀도는 높이고, 핵심 행동은 한 화면에 유지한다.
- 검증된 공덱과 상태 정보는 색상만으로 구분하지 않고 라벨과 아이콘을 함께 사용한다.
- 어두운 작전실 분위기를 기본으로 하되 본문 대비와 터치 영역을 우선한다.

## 색상 토큰

| 역할 | 토큰 | 값 |
| --- | --- | --- |
| 전체 배경 | `background-base` | `#0B1020` |
| 사이드바·상단바 | `background-elevated` | `#131A2B` |
| 카드·입력 영역 | `surface` | `#192238` |
| 호버 영역 | `surface-hover` | `#202C47` |
| 기본 테두리 | `border` | `#2A3652` |
| 주요 텍스트 | `text-primary` | `#F4F7FB` |
| 보조 텍스트 | `text-secondary` | `#AAB5C8` |
| 길드·강조 | `accent-gold` | `#E7B85C` |
| 주요 행동 | `accent-blue` | `#4E8BFF` |
| 성공·검증 | `success` | `#38C78B` |
| 위험·삭제 | `danger` | `#F0646A` |

속성색은 불 `#E85D5D`, 물 `#4E8BFF`, 바람 `#D5A83E`, 빛 `#E9DDA8`, 어둠 `#8A6BD1`을 사용한다.

## 타이포그래피와 형태

- 글꼴: `Noto Sans KR` 우선, 이후 Apple SD Gothic Neo, 맑은 고딕, 시스템 글꼴 순서
- 본문: 15px / 1.5
- 페이지 제목: 28–32px / 1.25
- 카드 모서리: 12px
- 모달 모서리: 16px
- 최소 터치 영역: 44×44px
- 키보드 포커스: 파란색 3px 외곽선과 2px 간격

## 레이아웃

- 데스크톱: 240px 고정 사이드바와 유동 콘텐츠 영역
- 900–1199px: 검색 결과를 1열로 전환하고 콘텐츠 폭을 확보
- 768–899px: 사이드바를 숨기고 모바일 헤더와 하단 탭 사용
- 767px 이하: 선택 영역과 카드 메타를 세로 배치, 몬스터 선택창을 전체 화면으로 확장

## 핵심 컴포넌트

- `AppShell`: 길드 전환, 주요 탐색, 알림, 프로필, 모바일 하단 탭
- `MonsterPortrait`: 속성 링, 등급, 선택 순서, 리더 표시를 조합
- `MonsterPicker`: 이름 검색, 속성 필터, 정확히 3마리 선택, 확인/취소
- 상대 방어덱의 개별 몬스터 카드를 누르면 해당 슬롯만 즉시 교체하며, 다른 슬롯과의 중복 선택은 막는다.
- `MonsterArchive`: ryhlab 기준 845마리 도감, 한국어 약칭·영문 각성명 검색, 속성·태생 등급 필터
- `DeckCard`: 방어덱과 공격덱 비교, 승률·사용 횟수·최근 사용, 공식 공덱 상태
- `AccessGate`: 닉네임·길드 코드 접근 요청, 승인 대기, 반려 후 재요청 상태
- `RequestInbox`: OFFICER/OWNER용 요청 검토, 승인·반려, 최근 처리 내역
- `GuildHome`: 공지·일정·숙제를 우선순위 순서로 요약하고 각 작업으로 바로 이동
- `TeamBuildEditor`: 3–15마리 편성, 슬롯 교체, 리더·룬·9개 스탯·메모를 같은 패턴으로 입력
- `DungeonGuides`: 분류 탭과 전체 검색, 던전별 권장 편성 인원, 공략 상세를 한 화면에 배치
- `HomeworkBoard`: 작성 권한을 서버에서 검증하고 길드원별 완료 상태를 독립적으로 표시
- 버튼: Primary는 금색, Secondary는 표면색, 완료 행동은 초록색, 위험 행동은 빨간색을 사용

## 상태와 접근성

- 로딩은 레이아웃을 유지하는 스켈레톤을 사용한다.
- 빈 결과는 선택 조건을 유지하고 재검색 방법을 안내한다.
- 오류는 원인과 재시도 행동을 함께 제공한다.
- `prefers-reduced-motion`을 존중하며, 애니메이션은 정보 이해를 돕는 경우에만 사용한다.
- 모든 아이콘 버튼은 접근 가능한 이름을 제공하고, 선택 상태는 `aria-pressed`로 노출한다.

구현 토큰의 원본은 `src/app/globals.css`, 컴포넌트 사용 예시는 `src/components`에서 확인할 수 있다.
